# 待完成清单 
                                                                                                                           
  一、交互功能（Phase 2 未完成部分）                                                                                                        
                                                                                                                                            
  缺失的规格要求（文档 §4.4）：
  - 边 hover 标签 — 鼠标悬停在边上显示关系类型文字（"演化自" / "同时期代表作"），目前无此交互
  - 节点 hover 卡片 — 当前用的是浏览器原生 <title> tooltip，规格要求显示菜肴名 + 时期的富文本预览卡
  - 演化链高亮 — 点击选中节点后，其直接前身和演化后裔应一并高亮（区别于 hover 的整体高亮），目前只做了 hover 效果
  - 时期清除按钮 — 切换到某个时期后，没有清晰的方式返回全图视角（点击当前时期会切换但 UX 不明显）

  ---
  二、视觉与动画

  - Remy 角色图 — 当前用 🐭 emoji 占位；规格描述"巴黎屋顶飞入画面"的入场动画需要实际角色图（PNG 透明底）
  - 图谱过渡动画 — 切换时期时图谱直接重绘，无平滑过渡；节点应有 fade-in / scale-in 入场
  - 全图标签密度 — 全部 35 个节点同时显示时，法语名标签互相遮挡，需要按缩放层级控制标签显示
  - 窗口 resize 响应 — D3 画布尺寸硬编码为初始 clientWidth/Height，调整窗口大小后图谱不自适应

  ---
  三、数据与内容

  - 历史准确性校验 — 35 道菜肴内容为基于文档框架手写，尚未与 Larousse Gastronomique / Wikidata 交叉验证（文档 §5.3 要求人工标注两周）       
  - 演化关系补全 — 规格列出的演化路径（如 Vol-au-vent → Feuilleté，Blanc manger → 当代 Blancmange）有部分目标节点不在当前数据中
  - Wikidata 自动化抓取 — 文档 Phase 0 要求通过 SPARQL 批量获取节点基础信息，目前全为手写
  - Remy 旁白语言 — 文档指定 Remy 开场白为法语（语气 A），当前写的是中英混合；需与内容策略对齐

  ---
  四、技术优化

  - 移动端适配 — 当前三栏布局在小屏幕无法使用（文档 Phase 3 要求）
  - SSR 水合问题 — D3 通过动态 import('d3') 规避 SSR，但 SVG 初始尺寸为 0 可能造成短暂空白，需加 loading 状态
  - 图谱性能 — 切换时期时完全销毁重建 D3 实例，可优化为增量更新（保留 zoom 状态）
  - 无障碍 — 图谱节点缺少 aria-label，键盘导航不可用

  - 基础埋点（停留时长、节点点击率、"听 Remy 说"展开率——文档 §7.3 成功指标）
